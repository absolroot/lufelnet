# 번역
신규 파일/기능을 만들 때는 항상 /i18n 을 참조하여 각 언어별로 번역이 될 수 있도록 해야합니다.
번역 관련 작업(문구 번역, 고유 명사 매칭, 다국어 필드 추가, 기존 번역 검수)을 수행할 때는 항상 `p5x-translator` 스킬을 우선 사용해야 합니다.
게임 정보에 대한 사이트이기 때문에 고유 명사가 상당히 많고, 고유 명사를 임의로 의역하지 않고 불분명한 경우가 있는 경우 꼭 사용자에게 재질문 또는 정리를 통해 확인받아주세요.
현재 지원하는 언어는 `en`, `kr`, `jp`, `cn`입니다. 나중에 더 많은 언어를 추가할 수도 있어요.

# 서버 특징
- `kr`, `cn`, `tw`는 같은 서버 그룹으로 취급합니다.
- `en`, `jp`, `sea`는 같은 글로벌 서버 그룹으로 취급합니다.
- 출시 순서는 `cn`이 가장 빠르고, `kr`, `tw`는 `cn`보다 1주일 뒤에 출시됩니다.
- 글로벌 버전은 `en`, `jp`가 먼저 출시되고, `sea`는 그보다 6일 뒤에 출시됩니다.
- 따라서 언어/콘텐츠 로직을 묶을 때는 기본적으로 `kr`, `cn`을 함께 보고, `en`, `jp`를 함께 봅니다. `tw`, `sea`도 각각 같은 서버 그룹 로직을 따릅니다.
- `en`, `jp`, `sea`는 `kr`, `cn`, `tw`보다 늦게 출시된 글로벌 서버라 미출시 콘텐츠가 많습니다. 그래서 한국어/중국어 기준의 모든 내용이 글로벌에 그대로 존재하지 않을 수 있습니다.
- 문서, 주석, 설명, 메모를 남길 때 `KR only`, `KR 서버 전용`, `KR v4.7.1 출시`처럼 쓰지 말고, 실제 서버 기준에 맞춰 `KR/CN`, `KR/CN v4.7.1 출시`처럼 표기합니다. 글로벌 기준 설명도 같은 방식으로 `EN/JP`, 필요하면 `EN/JP/SEA`로 표기합니다.

# 필수 조건
- 코드 수정하는 과정에서 한국어나 일본어 등 다른 언어를 변환하다가 UTF 에러 등 깨지는 경우가 많은데 항상 이런 일이 없도록 점검할 것
- 즉 UTF-8 BOM 으로 저장하지 말것.
